---

## 一、参赛资格

- **参赛者**：不限学历、不限职业，个人或团队（≤5人）均可参加。
- **参赛承诺**：禁止使用真实患者数据；不得做出医疗诊断承诺；项目默认开源。

---

## 二、赛题方向

赛题方向为**通用医学 + 生命科学**方向，不做更细的限制。作品须满足以下条件：

1. **医疗场景导向**：聚焦医疗垂直领域（如肿瘤、罕见病、诊断辅助、患者管理、医学文献检索等）。
2. **技术形式**：形式为 **MCP 工具** 或 **Agent Skill**，可独立运行或集成到现有 Agent 框架。
3. **平台部署**：须能在魔搭社区部署为 Model/Space。
4. **选题登记**：选题不限细分方向，但需要在开发前在群内或活动页面**登记选题**，避免重复。

---

## 三、提交清单 (Checklist)

参赛者在魔搭活动页填写表单时，**必须包含**以下三个核心交付物：

| # | 交付物 | 说明 |
|---|--------|------|
| 1 | **魔搭社区作品链接** | 部署在魔搭的 Skill 或 MCP，确保公开可访问，有基本 Demo |
| 2 | **公开代码仓库地址** | 如 GitHub、Gitee 等，需含开源 License |
| 3 | **规范的 README.md** | 直接存放在代码仓库根目录，须满足下方模板要求 |

### 提交入口

- **Skill 提交**：[https://modelscope.cn/skills/create?template=custom](https://modelscope.cn/skills/create?template=custom)
- **MCP 提交**：[https://modelscope.cn/mcp/servers/create](https://modelscope.cn/mcp/servers/create)

> 提交后非反动信息即可发布至社区，赛后统一整理为赛果合集进行宣传。

## 四、仓库设置规范

| 要求 | 说明 |
|------|------|
| **可见性** | 仓库必须设置为 **Public（公开）** |
| **开源协议** | 仓库根目录必须包含 `LICENSE` 文件。建议 **MIT** 或 **Apache 2.0** |
| **代码整洁** | 敏感信息（API Keys、隐私数据）不可硬编码，需用 `.env` 隔离 |
| **医疗合规** | 禁止使用真实患者数据；须声明输出不作为最终医疗诊断依据 |

### 建议的仓库结构模板

```
your-project/
├── README.md              ← 项目说明（必须）
├── LICENSE                ← 开源协议（必须）
├── requirements.txt       ← Python 依赖（推荐）
├── .env.example           ← 环境变量模板（推荐）
├── .gitignore             ← Git 忽略规则（推荐）
├── src/                   ← 核心源代码
│   └── ...
├── tests/                 ← 测试用例（加分项）
│   └── ...
├── docs/                  ← 补充文档（可选）
│   └── screenshots/       ← 运行截图
└── demo/                  ← 演示脚本或数据（可选）
```

## 六、评审规则（双轨加权评分制）

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| **社区热度分** | 40% | 基于作品在魔搭社区的调用数与收藏数 |
| **专家评审分** | 60% | 由医疗专家与技术大咖从以下三大维度打分 |

### 专家评审三大维度

| 维度 | 关注点 |
|------|--------|
| **实用性** | 是否解决真实医疗场景问题？受众是否明确？ |
| **代码质量** | 架构清晰度、可维护性、安全合规 |
| **文档完整性** | README 质量、部署说明、演示效果 |

---

## 七、专家评委额外加分项 (Bonus)

如果参赛者能提供以下内容，将在"专家评审分"中获得额外优势：

- ✅ **测试用例**：提供自动化测试覆盖核心功能。
